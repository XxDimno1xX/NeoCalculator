## Retired / superseded mappings

- MEM-1 (draw buffer rules) → unchanged, ratified in policy §1.1/§1.3; still no automated guard (HIL gap).
- MEM-2 (arena misuse) → split into MEMX-04/05/06/07 with concrete defects.
- MEM-3 (PSRAM exhaustion by sims) → corrected sizes; superseded by MEMX-09/11/12 (sims themselves are bounded and well-behaved; Python 1 MB and Giac are the real drivers).
- MEM-4 (eager `begin()` exhausting LVGL heap) → root-caused by MEMX-01 (the heap is 64 KB internal, which is *why* eager begin died).
- MEM-5 (emulator allocator divergence) → MEMX-13, now with the exact divergence list.
- MEM-6 (`EquationsApp::end()` steps.clear) → verified present (`EquationsApp.cpp:381-464`); residual risk folded into MEMX-01 growth vectors.
